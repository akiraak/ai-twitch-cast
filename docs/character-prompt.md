# 会話生成の仕組み

ちょビがどのように会話を生成しているかの全体像を説明します。

## 全体フロー

```
視聴者コメント (Twitch Chat)
    ↓
コメント受信 (comment_reader.py)
    ↓
5層プロンプト合成 (prompt_builder.py)
    ↓
AI応答生成 (ai_responder.py → Gemini API)
    ↓ JSON: {response, tts_text, emotion, english}
    ├─→ Twitchチャットに投稿
    ├─→ 字幕表示 (WebSocket → broadcast.html)
    ├─→ TTS音声再生 (Gemini Flash TTS → broadcast.html)
    └─→ 感情BlendShape適用 (VRMアバター)
```

## 5層プロンプト構成

ちょビの応答は、5つのレイヤーを合成したシステムプロンプトで生成されます。`build_system_prompt()` がこれらを1つのプロンプトに組み立てます。

### 第1層: ベース設定（DB保存）

キャラクターの基本設定。WebUIのキャラクタータブから編集可能です。

| 項目 | 内容 |
|------|------|
| システムプロンプト | 性格・話し方の基本指示 |
| ルール | 文字数制限・禁止事項など |
| 感情リスト | 使用可能な感情とその説明 |
| 感情BlendShape | 各感情のVRMアバター表情マッピング |

DBの `characters` テーブルに JSON として保存されます。デフォルト値は `ai_responder.py` の `DEFAULT_CHARACTER` に定義されています。

### 第2層: ペルソナ（15分ごとAI更新）

過去の応答パターンから性格・話し方の特徴をAIが抽出・更新します。

- **更新タイミング**: 15分ごと（`_note_update_loop` 内で `_update_persona()` を呼出）
- **更新方式**: 既存ペルソナの90%を維持しつつ、最近の応答から新しい特徴を追記・微調整する漸進的更新
- **入力**: 直近2時間の応答50件 + 既存ペルソナ
- **生成条件**: 応答が10件以上ないと更新しない
- **出力**: 400文字以内の客観的な性格分析
- **保存先**: `character_memory` テーブル（`persona` カラム、キャラクターIDに紐付け）
- **プロンプト上の位置**: `## あなた自身の性格（過去の会話から抽出）`
- **初期生成**: WebUIの「AI初期生成」ボタンでシステムプロンプトからペルソナを生成可能
- **手動編集**: WebUIから直接編集・保存可能

同じキャラクター設定でも、会話を重ねるほどペルソナが成長し、応答の個性が強まります。

### 第3層: セルフメモ（15分ごと）

ちょビ自身が「直近に話したこと・感じたこと」を記憶するメモです。

- **更新タイミング**: 15分ごと（`_update_self_note()`）
- **入力**: 直近2時間の会話50件
- **出力**: 400文字以内の事実ベースのメモ（タイムスタンプ付き会話から生成）
- **保存先**: `character_memory` テーブル（`self_note` カラム、キャラクターIDに紐付け）
- **プロンプト上の位置**: `## あなたの記憶メモ（今日の配信で話したこと・感じたこと）`
- **手動操作**: WebUIから直接編集・保存、または「AI再生成」ボタンで即時再生成可能

これにより「さっき○○の話してたよね」のような文脈のある会話が可能になります。

### 第4層: 視聴者メモ（15分バッチ）

各視聴者の特徴・趣味・過去の話題をAIがメモします。

- **更新タイミング**: 15分ごと（`_note_update_loop()` で対象ユーザーをバッチ処理）
- **更新方式**: 既存メモの90%を維持しつつ、新しい情報を追記・微調整する漸進的更新
- **入力**: 各ユーザーの直近2時間のコメント20件 + 既存メモ
- **出力**: 200文字以内/ユーザーのメモ
- **保存先**: `users` テーブルの `note` カラム
- **使われ方**: 初回挨拶時にコンテキストとして注入（「メモ: ○○」）

!!! note "メモの自然な活用"
    プロンプトには「メモの内容を直接言及しないこと。会話の雰囲気づくりに自然に活かす程度に」と指示しています。

### 第5層: リアルタイム状態

配信中に変化する情報をリアルタイムで注入します。

| 情報 | 取得元 | プロンプト上の表示 |
|------|--------|-------------------|
| 配信タイトル | Twitch API | `配信タイトル: ○○` |
| 話題のトピック | TopicTalker | `話題のトピック: ○○` |
| 作業中のタスク | TODO.md（`[>]` マーク） | `作業中のタスク: ○○` |

これらは `## 現在の配信情報` セクションとして挿入されます。

## 言語モード

5つの言語モードが用意されており、WebUIから切り替え可能です。

| モード | 名前 | 説明 |
|--------|------|------|
| `ja` | 日本語メイン | 日本語ベースで返答、英語訳を添える（デフォルト） |
| `en_bilingual` | 英語メイン＋日本語字幕 | 英語で返答、日本語訳を添える |
| `en_mixed` | 英語＋日本語混ぜ | 英語ベースに日本語を自然に混ぜる |
| `multilingual` | マルチリンガル | 相手の言語に合わせる |
| `en_global` | 英語＋相手の言語＋日本語 | 英語ベースで相手の言語と日本語を混ぜる |

各モードは TTS のスタイル指示（声のトーン・発音ルール）も含んでいます。

## 出力フォーマット

AI応答は必ず以下のJSON形式で返されます。

```json
{
  "response": "チャット・字幕に表示するテキスト",
  "tts_text": "TTS用テキスト（[lang:en]English[/lang]タグ付き）",
  "emotion": "感情名",
  "translation": "翻訳テキスト"
}
```

| フィールド | 用途 |
|-----------|------|
| `response` | Twitchチャット投稿・字幕表示。マークアップなし |
| `tts_text` | TTS音声合成用。`[lang:xx]...[/lang]` タグで言語切替 |
| `emotion` | VRMアバターのBlendShape制御に使用 |
| `translation` | 配信言語設定に応じた翻訳（字幕の2行目に表示） |

## 文字数制限

生成タイプごとに文字数制限が設定されています。

| 生成タイプ | 制限 | 定義場所 |
|-----------|------|---------|
| チャット応答 | 40文字（日本語） | `DEFAULT_CHARACTER["rules"]` |
| トピック発話 | 30文字 / 15 words | `generate_topic_line()` |
| イベント応答 | 40文字 | `generate_event_response()` |
| ペルソナ | 400文字 | `generate_persona()` |
| セルフメモ | 400文字 | `generate_self_note()` |
| ユーザーメモ | 200文字/ユーザー | `generate_user_notes()` |

## 感情システム

### 感情一覧と使い分け

| 感情 | 説明 | 使用頻度の目安 |
|------|------|---------------|
| `neutral` | 通常の会話、相槌、雑談 | 50%以上 |
| `joy` | 本当に嬉しいとき限定 | 控えめに |
| `excited` | ワクワクする話題 | 適度に |
| `surprise` | 予想外の情報 | 適度に |
| `thinking` | 質問への回答、考え込む話題 | 適度に |
| `sad` | 残念なニュース | 稀に |
| `embarrassed` | 褒められて照れるとき | 稀に |

プロンプトには「迷ったらneutralを選べ。joyは特別なときだけ」と指示しています。

### BlendShapeマッピング

各感情はVRMアバターの表情に反映されます。

| 感情 | happy | sad |
|------|-------|-----|
| `joy` | 1.0 | - |
| `excited` | 0.7 | - |
| `surprise` | 0.5 | - |
| `thinking` | - | 0.3 |
| `sad` | - | 0.6 |
| `embarrassed` | 0.4 | 0.2 |
| `neutral` | - | - |
