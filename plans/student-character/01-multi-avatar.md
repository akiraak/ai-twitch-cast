## 残タスク: WebUI設定のマルチキャラクター対応

### 現状の問題

キャラクター関連の設定が3タブに散らばっている:

```
現在のタブ順: キャラクター → 会話モード → 配信画面 → ...

キャラクタータブ
├── ビジュアル サブタブ
│   └── ライティング（全アバター共通1つ、プリセット機能付き）
└── セリフ サブタブ
    ├── 配信言語
    ├── 第1層ベース設定（名前・プロンプト・ルール・感情・BlendShape）
    ├── 第2〜5層
    ├── 発話設定（文字数上限）
    └── テスト再生
    ※ 先生（ちょビ）1人分しかない

会話モードタブ
├── 教師モード（授業コンテンツ管理）
└── 雑談モード
    ※ キャラクター選択の概念がない

配信画面タブ
├── 背景
├── アバター（先生）  ← VRM選択 + スケール
├── アバター（生徒）  ← VRM選択 + スケール
├── 字幕 / TODO / カスタムテキスト...
    ※ VRM選択はキャラクターのビジュアルなのに配信画面にある
```

**問題点:**
1. **キャラクタータブが1人前提** — 先生しか設定できない。生徒を追加する場所がない
2. **VRM選択が配信画面タブにある** — キャラクターのビジュアルなのに場所が違う
3. **ライティングが全アバター共通** — 先生と生徒で別のライティングにしたい
4. **会話モードにキャラクター概念がない** — 教師モードは先生+生徒で授業するのにキャラクター選択がない

### 設計: モード主導の構成

**基本方針**: 「今何をしているか（モード）」が操作の起点。キャラクターはモードの構成要素。

```
タブ順: 会話モード → キャラクター → 配信画面 → ...
```

- **会話モード** = 「今何をしているか」（配信中に一番触る、操作の起点）
- **キャラクター** = 「誰がいるか」（キャラの定義・設定）
- **配信画面** = 「どう見えるか」（レイアウト・配置の微調整）

```
会話モードタブ（操作の起点 — 配信中に一番触る）
├── 教師モード
│   ├── 登場キャラクター      ← [先生: ちょビ ✓] [生徒: まなび ✓]
│   │   └── キャラ名クリック → キャラクタータブへ遷移
│   └── 授業コンテンツ管理（既存）
└── 雑談モード
    └── 応答キャラクター      ← [ちょビ ✓]（現状は先生のみ）

キャラクタータブ（キャラの定義 — 配信前に設定する）
├── [先生（ちょビ）▼ / 生徒（まなび）] ← キャラクター切替セレクタ
│
├── ビジュアル サブタブ
│   ├── VRMファイル選択        ← 配信画面タブから移動
│   └── ライティング            ← キャラクターごとに独立
│       └── プリセット機能
│
└── セリフ サブタブ
    ├── 第1層ベース設定（名前・プロンプト・ルール・感情・BlendShape）
    ├── 第2〜5層（先生のみ。生徒は第1層のみ）
    ├── TTS設定（声・スタイル）  ← 新規追加
    ├── 発話設定（文字数上限）
    └── テスト再生
    ※ 配信言語は共通設定なのでキャラ切替の外（タブ上部 or 別の場所）

配信画面タブ（レイアウト — たまに微調整する）
├── 背景
├── アバター（先生）  ← 位置・サイズのみ（VRM選択はキャラクタータブに移動済み）
├── アバター（生徒）  ← 位置・サイズのみ
├── 字幕
├── TODO
└── カスタムテキスト...
```

### 会話モードタブが起点になる理由

配信中の操作フロー:
1. **会話モードタブ**で「教師モードで授業をする」と決める
2. 教師モード内に「登場キャラクター: 先生✓ 生徒✓」が見える
3. 生徒の設定を変えたい → キャラ名をクリック → **キャラクタータブ**に遷移（生徒が選択された状態で）
4. VRM・ライティング・セリフを調整
5. 位置を変えたい → **配信画面タブ**でドラッグ調整

会話モードタブからキャラクタータブへの導線があることで、散らばっている感覚がなくなる。

### キャラクター切替セレクタ

キャラクタータブの上部に配置:

```
┌──────────────────────────────────────┐
│  [● 先生（ちょビ）] [  生徒（まなび）]   │
├──────────────────────────────────────┤
│  ビジュアル | セリフ                      │
│  ...                                   │
```

- 切替時にDB/APIから該当キャラクターの設定を読み込み
- 先生: 既存の `characters` テーブルのデータ
- 生徒: `settings` テーブルの `student.*` キー（Step 4 で追加予定）、または `characters` テーブルに2行目追加
- 会話モードタブのキャラ名クリックで遷移した場合、該当キャラが選択された状態で開く

### ライティングの個別化

**現状**: `window.avatarLighting` の `setExposure` / `setColor` が全インスタンスに一括適用

**変更**:
- 各 `AvatarInstance` が自身のライティング状態を持つ（既に実装済み: `this.ambientLight`, `this.dirLight`）
- DB: `lighting.avatar1.*` / `lighting.avatar2.*` で個別保存
- `_applyLighting(lighting, avatarId)` でアバター指定適用
- WebUI: キャラクター切替セレクタに連動してライティングスライダーの値が切り替わる
- プリセット: キャラクターごとに保存 or 共通プリセットをキャラクターに適用する選択式

### 実装順序

1. **タブ順変更**（会話モード → キャラクター → 配信画面）+ 初期表示を会話モードに
2. **キャラクター切替セレクタUI追加**（キャラクタータブ上部、先生/生徒の切替）
3. **VRM選択をキャラクタータブに移動**（配信画面タブのアバターパネルからVRM部分を削除）
4. **ライティングDB・APIをアバター個別化**（`lighting.*` → `lighting.avatar1.*` / `lighting.avatar2.*`）
5. **ライティングUIをキャラクター切替に連動**（切替時にスライダー値を差し替え）
6. **配信画面タブのアバターパネルを位置・サイズのみに整理**
7. **会話モードタブに登場キャラクター設定を追加**（教師モード内、キャラクタータブへの遷移リンク付き）
