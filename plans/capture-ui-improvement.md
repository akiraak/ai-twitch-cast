# WebUIのウィンドウキャプチャ管理表示の改善

## ステータス: 完了

## 背景

現在の「配信画面」タブのウィンドウキャプチャ管理UIには以下の課題がある：

1. **アクティブキャプチャの情報が最小限** — ラベルと「停止」ボタンしかない
2. **保存済み設定がDebugタブに隠れている** — 普段使うには不便
3. **復元ボタンがUI上にない** — 保存済み設定の復元はAPIのみ
4. **表示/非表示の切り替えがない** — broadcast.htmlでしか操作できない
5. **キャプチャのレイアウト設定が配信画面セクションの下に長く続く** — アバター・字幕・TODO・トピックと同じ場所に並び、キャプチャ数が増えると見づらい

## 改善方針

### Step 1: アクティブキャプチャ一覧の強化

**対象ファイル**: `static/js/index-app.js` (`captureRefreshSources`関数)

現在の表示:
```
[ラベル] [停止ボタン]
```

改善後:
```
[ラベル]                         [表示/非表示] [停止]
  ウィンドウ名: xxxxx  レイアウト: X:5 Y:10 W:40 H:50
```

変更内容:
- ウィンドウ名（`c.name`）を2行目に小さく表示
- レイアウトサマリー（X/Y/W/H）を表示
- 表示/非表示トグルボタンを追加（`layout.visible`を切り替え）
- 停止ボタンに確認ダイアログ追加（既存の`showConfirm`を使用）

### Step 2: 保存済みキャプチャをメインUIに統合

**対象ファイル**: `static/index.html`, `static/js/index-app.js`

変更内容:
- DebugタブからキャプチャカードへWordの保存済みキャプチャを移動
- アクティブキャプチャ一覧の下に「保存済み設定」セクションを追加
- 「一括復元」ボタンを追加（既存の`captureRestoreAll`を使用）
- 各保存済み設定に個別の「復元」ボタンも追加
- Debugタブの保存済みキャプチャカードは削除

### Step 3: キャプチャレイアウト設定の折りたたみ化

**対象ファイル**: `static/js/index-app.js`, `static/css/index.css`

変更内容:
- 各キャプチャのレイアウトfieldsetを折りたたみ可能にする（`<details>`/`<summary>`を使用）
- デフォルトは閉じた状態
- fieldsetの代わりに `<details>` で囲む

### Step 4: CSSの整理

**対象ファイル**: `static/css/index.css`

変更内容:
- キャプチャ一覧アイテム用のスタイルクラスを追加（インラインスタイルからの移行）
- 保存済み設定セクション用のスタイル追加
- 表示/非表示トグルボタンのスタイル

## 影響範囲

- `static/index.html` — HTMLテンプレートの変更
- `static/js/index-app.js` — JS関数の修正
- `static/css/index.css` — スタイル追加
- `scripts/routes/capture.py` — 変更なし（既存APIで対応可能）

## リスク

- broadcast.htmlへの影響なし（WebSocket通信は変更しない）
- 既存APIは変更しない（フロントエンドのみの変更）
- レイアウト設定のslider動作はそのまま維持
