## v2で実装する機能

### 機能1: URLテキスト抽出の改善

**優先度: 高** / **難易度: 低**

**現状の問題**:
- `extract_text_from_url()` はhttpxで生HTML取得→先頭30,000文字をGeminiに丸投げ
- 広告・ナビゲーション・スクリプト等のノイズが多く、Geminiのトークンを浪費
- 抽出品質が不安定

**改善内容**:
- BeautifulSoup（requirements.txtに既存）でHTML前処理
- `<script>`, `<style>`, `<nav>`, `<footer>`, `<iframe>`, `<aside>` 等を除去
- `<article>`, `<main>`, `.entry-content`, `.post-content` 等のセマンティック要素を優先抽出
- クリーンなテキストをGeminiに送信（トークン効率向上+抽出精度向上）

**変更対象**:
- `src/lesson_generator.py` の `extract_text_from_url()` のみ

---

### 機能2: マルチURLソース対応

**優先度: 高** / **難易度: 中**

**現状の問題**:
- URLを追加すると `_clear_lesson_data()` で全ソース（画像含む）が削除される
- 1つのコンテンツに複数URLの情報を統合できない
- DBスキーマ上は複数ソース対応済みだがAPIとUIが制限している

**改善内容**:
- URL追加時に既存ソースを消さず追加する（画像追加と同じ挙動に統一）
- テキスト抽出を「全ソースから再抽出」ボタンに変更
  - 画像: 各画像からGeminiでテキスト抽出
  - URL: 各URLからBeautifulSoup+Geminiでテキスト抽出
  - 結果を結合して `extracted_text` に保存
- ソース一覧で個別削除が可能（現状でもUIにある）
- 管理画面のURL入力UIを改善（画像と同じくソース追加フロー内）

**変更対象**:
- `scripts/routes/teacher.py` — `add_url` エンドポイントの `_clear_lesson_data()` 呼び出し除去、`extract_all_text` エンドポイント追加
- `static/js/admin/teacher.js` — ソース追加UI修正、テキスト再抽出ボタン
- `src/lesson_generator.py` — 複数ソース統合のテキスト抽出関数

---

### 機能3: 授業テキストパネルの書式対応

**優先度: 中** / **難易度: 低**

**現状の問題**:
- `display_text` は `.textContent` でプレーンテキスト表示のみ
- 箇条書き・太字・コード等の構造を表現できない

**改善内容**:
- `display_text` に簡易Markdown記法を許可
  - `**太字**`, `- リスト`, `\n` 改行
- broadcast.htmlのパネルで軽量Markdownパース→HTML表示
- XSS対策: 許可タグのホワイトリスト制限

**変更対象**:
- `static/js/broadcast/panels.js` — `showLessonText()` でMarkdownパース
- スクリプト生成プロンプトに「display_textはMarkdown記法可」を追記

---

### 機能4: 授業中のチャット割り込み改善

**優先度: 中** / **難易度: 中**

**現状の問題**:
- 授業中にチャットコメントが来ても、授業が止まらず並行して発話が衝突する可能性がある
- 問いかけセクションの待ち時間中にチャットが来ても、授業の文脈で応答されない場合がある

**改善内容**:
- 授業中のチャット応答を一時停止対応にする（SpeechPipelineのロックで排他）
- questionセクションの待ち時間中にチャットが来たら、その内容を回答に反映するオプション

**変更対象**:
- `src/lesson_runner.py` — チャット割り込み検知
- `src/comment_reader.py` — 授業中の応答優先度

---

## 推奨実装順序

1. **URLテキスト抽出改善**（既存関数の内部変更のみ、低リスク）
2. **マルチURLソース対応**（API+UI変更、テキスト抽出改善が前提）
3. **授業テキストパネル書式対応**（フロントエンドのみ、独立）
4. **チャット割り込み改善**（バックエンド変更、要設計）

## リスク

- BeautifulSoupのセレクタが特定サイトで効かない場合がある → フォールバックとして現行のGemini直接抽出を残す
- マルチURL統合時にテキスト量が膨大になる可能性 → 各ソースの抽出文字数に上限設定
- Markdownパースで意図しないHTML出力 → ホワイトリスト方式でサニタイズ
