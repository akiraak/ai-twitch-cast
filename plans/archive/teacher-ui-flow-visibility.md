# 教師モード管理画面 — 全生成フロー可視化

## ステータス: 検討中

## 目的

管理画面の教師モードで、**すべての処理の流れとそこに使われているデータ**が追えるようにする。
「なぜこの結果になったのか」を各ステップの入力・プロンプト・出力から辿れる状態にする。

---

## 全体フロー（現状の処理）

```
Step 1: ソース追加
  画像/URL → [Gemini Vision/URL解析] → extracted_text

Step 2a: プラン生成（3エキスパート）
  2a-1: 知識先生
    入力: extracted_text + 画像
    プロンプト: 「教材を分析し、教えるべき要点・学習順序・よくある誤解を出せ」
    出力: knowledge（テキスト）

  2a-2: エンタメ先生
    入力: extracted_text + 画像 + knowledge（知識先生の出力）
    プロンプト: 「知識先生の分析を元に、起承転結の物語構造を設計せよ」
    出力: entertainment（テキスト）

  2a-3: 監督
    入力: knowledge + entertainment（教材テキスト・画像は渡さない）
    プロンプト: 「両者の提案を統合し、セクション構成をJSON配列で出力せよ」
    出力: plan_sections（JSON配列）

Step 2b: スクリプト生成
  入力: plan_sections + extracted_text + 画像 + キャラ設定（先生・生徒）
  プロンプト: 「プランに従いスクリプトを生成。対話形式でdialogues配列を含める」
  出力: sections（JSON配列）— content, tts_text, display_text, dialogues

Step 2c: TTS音声生成
  入力: sections の各発話 + キャラのvoice/style
  処理: 各発話ごとに Gemini TTS API → WAVファイル
  出力: resources/audio/lessons/{id}/{lang}/section_XX_dlg_YY.wav

Step 3: 授業再生
  入力: sections + WAVファイル
  処理: LessonRunner → speech_pipeline → C#アプリ送信
```

## 改善案

### 設計方針

**各ステップの「入力 → 処理 → 出力」を明示的に表示する。** 折りたたみで詳細を隠しつつ、展開すれば全データが追える。

---

### Step 1: ソース → 教材テキスト

現状で十分。軽微な追加のみ:
- 抽出方法バッジ（`画像OCR` / `URL抽出` / `手動入力`）
- 抽出テキストの文字数表示

## プロンプト表示の実装方法

各エキスパート・スクリプト生成のプロンプトは長大（数百行）。以下の方法で対応:

1. **APIにプロンプト取得エンドポイントを追加**: `GET /api/lessons/{id}/prompts?step=knowledge&lang=en`
   - lesson_generator.py からプロンプト生成ロジックを呼び出し、実際に使われるプロンプトを返す
   - 実際のLLM呼び出しは行わない

2. **管理画面**: 折りたたみ内に `<pre>` でプロンプト全文を表示。[コピー]ボタン付き

---

## 実装の優先順位

1. **Step 2a: 3エキスパートのフロー可視化** — 入力→プロンプト→出力のデータフローを明示。エキスパート間の依存関係の矢印表示
2. **Step 2b: スクリプト生成の入力表示** — キャラ設定・プロンプト・生成方法の明示
3. **Step 2c: TTS生成のStep分離** — 各発話のvoice/style/キャッシュ状態を明示
4. **Step 3: 再生状態の改善** — 再生中セクションのハイライト、音声ステータス
5. **プロンプト表示API** — 各ステップのプロンプト全文取得
