# 授業モード v3 — 手動テストプラン

## 前提条件

- サーバー起動済み（`./server.sh`）
- 管理画面にアクセス可能（`http://localhost:8080`）
- 自動テスト通過済み（`python3 -m pytest tests/ -q` — 573 passed）

---

## Phase 1: プラン生成の検証

短い教材（画像 or URL）でプラン生成し、監督主導アーキテクチャが正しく動作するか確認する。

### 1-1. コンテンツ作成 → ソース追加
- [ ] 新規コンテンツを作成
- [ ] 画像またはURLをソースとして追加
- [ ] （画像の場合）テキスト抽出が完了する

### 1-2. プラン生成（日本語）
- [ ] 「プラン生成」ボタンを押す → SSE進捗表示が出る
- [ ] 生成完了トーストが表示される（「プラン生成完了 (Nセクション構成)」）
- [ ] エラーなく完了する

### 1-3. 監督出力（director_sections）の品質確認
- [ ] `display_text` が具体的な教材内容を含んでいる（抽象的な「今日のテーマ」だけでない）
- [ ] `dialogue_directions` が十分に具体的（2〜3文の指示になっている）
- [ ] `key_content` が教材の要点を網羅している
- [ ] セクション数が適切（3〜15）

---

## Phase 2: スクリプト生成 + TTS の検証

### 2-1. スクリプト生成
- [ ] 「スクリプト生成」ボタンを押す → SSE進捗表示が出る
- [ ] 生成完了トーストが表示される（セクション数・TTS件数）
- [ ] 各セリフが監督の `dialogue_directions` の指示に沿っている
- [ ] `key_content` で指定された内容がセリフに含まれている

### 2-2. TTS再生
- [ ] 各セリフの ▶ ボタンで音声が再生される
- [ ] セクション単位の ▶ ボタンで連続再生される
- [ ] 音声が自然に聞こえる

---

## Phase 3: 管理画面 UI の検証（Step 7）

### 3-1. Phase A: 教材分析カード
- [ ] 「📘 Phase A: 教材分析」ヘッダーが表示される
- [ ] Step 1（知識先生）: モデル名・temperatureが表示される
- [ ] Step 1: システムプロンプトが折りたたみで全文表示される（展開可能）
- [ ] Step 1: ユーザープロンプトが折りたたみで全文表示される
- [ ] Step 1: 出力がデフォルト展開で全文表示される
- [ ] Step 2（エンタメ先生）: 同上の構造
- [ ] Step 3（監督）: 同上の構造 + 生出力（JSON）が折りたたみで表示

### 3-2. データフロー可視化
- [ ] Step 1→2 間に「▼ 知識先生の出力が入力に含まれる」の矢印がある
- [ ] Step 2→3 間に「▼ 知識 + エンタメの出力が入力に含まれる」の矢印がある

### 3-3. 監督パース結果
- [ ] 「📋 パース結果: Nセクション」が表示される
- [ ] 各セクションが展開可能カードで表示される
- [ ] `display_text` が表示される
- [ ] `dialogue_directions` が speaker/direction/key_content の構造で表示される
- [ ] 番号表記（例: [1-1], [1-2]）でセリフとの対応が分かる

### 3-4. Phase C: セリフ個別生成
- [ ] 「🎭 Phase C: セリフ個別生成」ヘッダーが表示される
- [ ] 各セリフカードに「🎬 監督: (指示内容)」が薄紫背景で表示される
- [ ] `key_content` が表示される
- [ ] 各セリフの「🔍 生成プロンプト」を展開 → model/temp/システムプロンプト/ユーザープロンプト/生出力が全文確認できる

### 3-5. プロンプトがAPIから取得されている確認
- [ ] **検証方法**: ブラウザDevToolsでNetwork → `/api/lessons/{id}` のレスポンスに `plan_generations` が含まれている
- [ ] `plan_generations` 内に `knowledge.system_prompt`, `entertainment.system_prompt`, `director.system_prompt` がある
- [ ] JSソースコードに「知識先生」「エンタメ先生」「監督」のシステムプロンプト本文がハードコードされていない

### 3-6. データ永続化
- [ ] ページリロード後も全データが保持されている
- [ ] Phase A のプロンプト・出力が消えていない
- [ ] Phase C の生成メタデータが消えていない

---

## Phase 4: 授業再生の検証

### 4-1. 授業開始
- [ ] 「授業開始」ボタンを押す → 授業が開始される
- [ ] broadcast.html（配信プレビュー）で表示される

### 4-2. 授業の流れ
- [ ] セクションが順番に再生される
- [ ] display_text が画面に表示される
- [ ] キャラクターが自然に対話している
- [ ] 感情がセリフに合っている

### 4-3. 授業制御
- [ ] 一時停止 → 再開が機能する
- [ ] 終了ボタンで停止する

- [ ] 言語タブを English に切り替え
- [ ] プラン生成 → 英語のプロンプト・出力が表示される
- [ ] スクリプト生成 → 英語のセリフが生成される

## 判定基準

| 項目 | 合格条件 |
|------|---------|
| プラン生成 | エラーなく完了、director_sectionsに具体的内容 |
| スクリプト生成 | 監督の指示に沿ったセリフ、key_content反映 |
| 管理画面UI | 全LLM入出力が折りたたみで全文確認可能 |
| データフロー | ステップ間の矢印・番号紐付けが正確 |
| 永続化 | リロード後もデータ保持 |
| 授業再生 | 自然な流れで再生完了 |
