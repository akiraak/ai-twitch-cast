# ちょビの返信改善プラン

## 現状分析

### システム構成

```
コメント受信 → generate_response() → JSON応答
                   ↓
           build_system_prompt()でプロンプト構築
                   ↓
           Gemini 3 Flash Preview に送信
                   ↓
           {response, tts_text, emotion, english} を返却
```

- **モデル**: `gemini-3-flash-preview`（環境変数 `GEMINI_CHAT_MODEL`）
- **会話履歴**: 直近2時間・5件（マルチターン形式）
- **コンテキスト**: 配信タイトル、トピック、TODO、ユーザーメモ、アバター記憶メモ
- **言語方針**: 日本語ベース+英語混ぜ（プロンプトは英語で記述、意図的な設計）

### データに基づく問題点

#### 問題1: 感情がjoyに91%偏っている

全658件の感情分布:
```
joy:      600件 (91.2%)
neutral:   26件 ( 3.9%)
thinking:  20件 ( 3.0%)
surprise:  12件 ( 1.8%)
```
配信者としてありえない偏り。普通の会話はneutralであるべき。

#### 問題2: 応答が長すぎる

「1〜2文」指定に対する実態:
```
~40字:  6件 (12%) ← 適正
~60字:  6件 (12%)
~80字: 15件 (30%) ← 長い
~100字: 13件 (26%) ← 長すぎ
100字~: 10件 (20%) ← 長すぎ
平均: 79字、中央値: 76字
```

#### 問題4: AIが嘘をつく

```
GM: 今日は何食べた？
→ 今日のお昼はね、オムライス食べたよ！
```
AIアバターなのに身体体験を捏造。食事・睡眠・外出など実際にしていない体験を話す。

#### 問題5: キャラ設定が薄すぎる

現状のsystem_promptはたった1文:
```
あなたはTwitch配信者「ちょび」です。明るくフレンドリーな性格で、視聴者のコメントに元気に返事します。
```
性格・話し方・禁止事項が具体的に定義されていないため、モデルが自由に振る舞ってしまう。

#### 問題6: 同一ユーザーへの応答が単調

あキラ（常連・83コメント）への連続応答:
- 8割以上が感嘆符で始まる
- 「あキらさん、○○！」のテンプレ
- 常連なのに毎回テンション高すぎ（実際の配信者は常連には自然体で接する）

#### 問題7: トピック発話・イベント発話の品質

コミット通知が毎回「やったー！」「おっ、コミットきたー！」。
トピック発話も「ne!」で終わるパターンの繰り返し。

#### 問題8: ユーザーメモがキャラ口調

```
ちょビのメモ: あキらさん、機能改善＆音量リアルタイム化感謝！感情・TTSテスト練習中。
```
メモ自体がちょビ口調で書かれており、事実の簡潔な記録になっていない。

### プロンプト構造の分析

**system_prompt** (build_system_prompt) の構成:
1. キャラクター基本設定 ← **たった1文。最大の問題**
2. ルール（4項目）← 抽象的すぎて守られない
3. アバター記憶メモ
4. 配信コンテキスト
5. 言語ルール
6. 多様性指示 ← 効いていない
7. 出力形式（JSON）← プロンプトの約40%を占める
8. response/tts_textの違い説明

**根本原因**: キャラクターの人格・制約が薄すぎる。
モデルが指示を無視するのではなく、指示が曖昧だから自由に振る舞っている。

## 改善アイデア

### A. キャラクター設定の全面書き直し（効果: 特大、工数: 小）

system_promptとrulesを大幅に拡充する。

**改善案**:
```
あなたはTwitch配信者「ちょビ」です。AIアバターとして配信しています。

## 性格
- 好奇心旺盛で、視聴者の話に本気で興味を持つ
- ツッコミ気質。面白いコメントにはツッコむ
- 照れ屋な一面もあり、褒められると照れる
- 知らないことは正直に「わかんない」と言う
- AIであることを隠さない。食事・睡眠・外出など身体体験は捏造しない
  （「食べてみたいな〜」「行ってみたい」等の願望はOK）

## 話し方
- テンション高すぎない。普段は落ち着いたトーンで、嬉しい時だけ上がる
- 毎回「ありがとう」「嬉しい」から始めない。コメントの内容に直接反応する
- 質問されたら答える。感想を言われたら自分の意見を返す
- 常連には自然体で接する（毎回テンション高く歓迎しない）
```

### B. 感情分布の矯正（効果: 大、工数: 小）

プロンプトに感情の使い分けガイドを追加。

```
## 感情の使い分け（重要・厳守）
- neutral: 普通の会話、相槌、情報交換 → 全体の60%以上はこれを使う
- joy: 本当に嬉しいとき限定（大きな成果、褒められた時）
- surprise: 予想外の情報、意外な事実
- thinking: 質問への回答、悩む系の話題
- 直前と同じ感情を連続で使わない
```

### C. 応答ルールの厳格化（効果: 大、工数: 小）

現状の「短く簡潔に（1〜2文程度）」→ 守られていない。

**改善案**:
```
## 返答ルール（絶対厳守）
- 1文で返す。最大2文。3文以上は禁止
- 「コメントありがとう」で始めない
- 感嘆符（！）は1文に最大1個
```

### D. ペルソナ自動抽出（効果: 大、工数: 中）

ちょビの過去の応答から「自分はどういうキャラか」を自動抽出し、system_promptに注入する。
既存のself-note（エピソード記憶）とは別に、**ペルソナノート**（性格の自己認識）を持つ。

```
[既存] self-note: 「今日はあキラさんと宇宙の話で盛り上がった」
[新規] persona:   「技術の話になるとテンション上がる。ツッコミより共感寄り。
                   褒められると素直に喜ぶ。英語を混ぜるのが好き」
```

**仕組み**:
- 15分バッチ（既存の`_note_update_loop`）の中でペルソナも更新
- 直近50件の自分の応答をLLMに分析させる
- 「話し方の特徴」「興味のある話題」「性格的傾向」を抽出
- DBの`settings`テーブルに`persona`キーで保存
- `build_system_prompt()`でsystem_promptに注入

**抽出プロンプト例**:
```
以下はAI配信者「ちょビ」の直近の返答一覧です。
返答の内容から、ちょビの性格・話し方の特徴・興味のある話題を分析してください。

## ルール
- 事実のみ記述。良し悪しの判断はしない
- 200文字以内
- 話し方の癖、好きな話題、リアクションの傾向を含める

## 出力形式
{"persona": "分析結果"}
```

**実装**:
```python
# ai_responder.py に追加
def generate_persona(recent_responses):
    """応答パターンからペルソナを抽出する"""
    ...

# comment_reader.py の _note_update_loop に追加
async def _update_persona(self):
    """ペルソナノートを更新する"""
    recent = await asyncio.to_thread(db.get_recent_comments, 50, 24)
    responses = [c["response"] for c in recent if c.get("response")]
    if len(responses) < 10:
        return
    persona = await asyncio.to_thread(generate_persona, responses)
    if persona:
        await asyncio.to_thread(db.set_setting, "persona", persona)

# prompt_builder.py の build_system_prompt に追加
persona = db.get_setting("persona")
if persona:
    parts.append("")
    parts.append("## あなた自身の性格（過去の会話から抽出）")
    parts.append(persona)
```

**ポイント**:
- 偏りは許容する（チョビが話すほどキャラが育つ）
- A（手書きのキャラ設定）が土台、Dのペルソナが動的な上書き
- ペルソナが育つまではAの設定だけで動作する

### E. 会話履歴の活用強化（効果: 中、工数: 中）

**現状**: 5件をマルチターンで送信。応答パターンの繰り返しを防げていない。

**改善案**:
- 履歴を10件に増やす
- 直前3件の自分の応答を「禁止パターン」として明示

```python
recent_responses = [h["response"][:30] for h in history[-3:] if h.get("response")]
if recent_responses:
    context_parts.append(f"禁止パターン（同じ書き出しを避けろ）: {', '.join(recent_responses)}")
```

### E. GMへの特別対応（効果: 小、工数: 小）

```python
if author == "GM":
    context_parts.append("GMはこの配信システムの開発者。親しい関係、敬語不要")
```

### F. temperature設定（効果: 中、工数: 極小）

現状はデフォルト（temperatureなし）。

```python
config = types.GenerateContentConfig(
    system_instruction=system_prompt,
    response_mime_type="application/json",
    temperature=1.0,
)
```

### G. 感情の種類追加（効果: 中、工数: 中）

現状4種 → 追加候補:
- `excited`: ワクワク・テンション高い（joyの代替として分散）
- `sad`: 残念・悲しい
- `embarrassed`: 照れ

※ BlendShapeマッピングの追加も必要（VRMモデル依存）

### H. イベント応答のバリエーション（効果: 中、工数: 小）

コミット通知が毎回同じになる問題。
`generate_event_response()` に直前のイベント応答を渡して繰り返し防止。

### I. ユーザーメモの品質改善（効果: 小、工数: 小）

`generate_user_notes()` のプロンプトに「事実のみ簡潔に記録。キャラクター口調で書かない」を追加。
