# 授業スクリプト 前置き表現の除去

## 背景

授業モードで生成されるセクション冒頭の発話に「画面を見てね」「画面に〜を並べたよ」「読み上げるね」のような**メタ前置き**が頻出していて、聞いていて不自然・形式的に感じる。

実データ調査結果（`data/app.db / lesson_sections`）:

- `dialogues[0].content` に前置き定型句を含むセクション = **55 / 67（82%）**
- 検出した前置きキーワード: `画面を見`, `画面に`, `画面を読`, `読み上げ`, `読みます`, `見てね`, `見てください`, `読むね`
- 例:
  - 「**じゃあ画面を見てね。** これがKenjiの自己紹介文。Hi. My name is Kenji ...」
  - 「**画面に自己紹介のパターンを並べてみたよ。** My name is から始まって ...」
  - 「**画面にパターンをまとめたよ。読み上げるね。** 自己紹介パターン。1, My name is ...」

## 根本原因

`prompts/lesson_generate.md:232` のセクション dialogue 生成ルールが、前置きを**明示的に指示している**。

```
- **各セクションの最初のdialogue（speaker: teacher）で、`display_text` の内容を省略せず全文読み上げること**。
  視聴者は画面と音声の両方で内容を理解するため、先生が画面に表示されている内容をすべて声に出して読む。
  導入の一言（「画面を見てね」「読み上げるよ」など）を添えてから全文を読み上げ、その後に解説や対話に入る。
  `display_text` が長い場合（教材本文など）でも省略しない
```

このルールには2つの要件が混在している:

| 要件 | 必要か | 現状 |
|------|-------|------|
| (A) `display_text` の中身を省略せず音声で読み上げる | **必要** — 視聴者は画面＋音声の両方で内容を理解する | 維持したい |
| (B) 「画面を見てね」「読み上げるよ」のメタ前置きを付ける | **不要** — 形式的・不自然 | 除去したい |

LLMは(B)を明示的に指示されているので機械的に毎セクション差し込み、結果として「またそれか」感が出ている。`prompts/lesson_improve.md` には同じ指示はないが、改善時にも前置きを温存しがち。

## 方針

1. **(A) を強化、(B) を除去**するようプロンプトを書き換える
2. **アンチパターン例**を明示して LLM が学習データの定型に流されないようにする
3. **良い例 / 悪い例**のペアで違いを具体化
4. 同じ修正を `prompts/lesson_improve.md` にも反映（改善時に前置きを再注入しないため）
5. 既存セクション 55件は **lesson_improver で順次フィードバック投入** または **再生成**で対処（即時の一括書き換えはしない）

## 採用スタイル: A（最小削除）

ユーザー判断: **「前置きを削るだけで、後ろの本文はそのまま」スタイル**で統一。

理由: 自然さ重視・全セクションで安定動作・プロンプト指示も短く済む。トピックフック挿入や会話調アレンジは LLM の揺れが大きく、前置きを別の形で復活させるリスクがあるため避ける。

## プロンプト書き換え（lesson_generate.md）

`prompts/lesson_generate.md:232` の1行を以下のブロックに置き換える。

```markdown
- **各セクションの最初のdialogue（speaker: teacher）で、`display_text` の内容を省略せず全文読み上げること**。
  視聴者は画面と音声の両方で内容を理解するため、先生が画面に表示されている内容を声に出して読む。
  `display_text` が長い場合（教材本文など）でも省略しない。

- **メタ前置きを入れない**。「画面を見てね」「画面を読むね」「読み上げるよ」「画面に〜を並べたよ」「画面の〜を読むね」のような、
  画面や読み上げ行為そのものに言及する一言は不要。視聴者は画面と音声を同時に追っているので、
  「これから読みます」という案内は冗長になる。**前置きを削った上で、後ろの教材内容はそのまま読み上げる**こと
  （別の言い回しで導入を作り直す必要はない）。

- 良い例 / 悪い例（前置きだけを削るスタイル）:

  | 種別 | ✕ 悪い | ◎ 良い |
  |------|--------|--------|
  | 導入 | **画面を見てね。** Unit 1: The First Week、英語で自分のことを紹介しよう! | Unit 1: The First Week、英語で自分のことを紹介しよう! |
  | 本文読み | **じゃあ画面を見てね。** これがKenjiの自己紹介文。Hi. My name is Kenji ... | これがKenjiの自己紹介文。Hi. My name is Kenji ... |
  | 本文読み | **では画面を見てね。Kenjiの自己紹介文を読み上げるよ。** Hi. My name is Kenji ... | Kenjiの自己紹介文。Hi. My name is Kenji ... |
  | 本文読み | **画面の本文を読み上げるね。** Hi. My name is Kenji ... | Hi. My name is Kenji ... |
  | 本文読み | **画面を読むね。** WRITING STRATEGY: When you take notes ... | WRITING STRATEGY: When you take notes ... |
  | 本文読み | **画面の言葉をそのまま読むね。** AIが得意なのは動くコードを書くこと。 | AIが得意なのは動くコードを書くこと。 |
  | パターン | **画面に自己紹介のパターンを並べてみたよ。** My name is から始まって、学校、授業 ... | 自己紹介のパターンは6つ。My name is から始まって、学校、授業 ... |
  | パターン | 次は質問するパターン。**画面に5つ並べたけど、まず** What country are you from? | 次は質問するパターン。5つあるよ。まず What country are you from? |
  | パターン | 次は質問の練習だよ。**画面を見てね。** Kenjiについて質問しよう。 | 次は質問の練習。Kenjiについて質問しよう。 |
  | クイズ | ここでクイズ！**画面を見て。** Kenjiのクラスはいつあるでしょう？ | ここでクイズ！Kenjiのクラスはいつあるでしょう？ |
  | クイズ | ここでクイズだよ。**画面を見てね。** 問題、Pierre について正しい文はどれ？ | ここでクイズ。Pierre について正しい文はどれ？ |
  | まとめ | **最後に画面のまとめを見てね。** 今日のまとめ。1. 自己紹介パターン ... | 今日のまとめ。1. 自己紹介パターン ... |
  | まとめ | **画面を読むね。** 今日の3つの持ち帰り。1、公開前に5分立ち止まる。 | 今日の3つの持ち帰り。1、公開前に5分立ち止まる。 |

- 導入として一言を入れたい場合も、画面・読み上げに言及せず**話題そのもの**で繋げる
  （例: 「次は質問するパターン。」「最後に大事な Culture Note。」「ここでクイズ！」など）。
  ただし**前置きを別の形で再注入しない**こと — 元の前置きを削るだけで自然に成立するなら、それで十分。
```

## `lesson_improve.md` への反映

`prompts/lesson_improve.md` の **ルール** セクション末尾に以下を追加:

```markdown
- セクション冒頭に「画面を見てね」「読み上げるよ」「画面に〜を並べたよ」のようなメタ前置きを入れない。
  教材内容にそのまま入ること（詳細は `prompts/lesson_generate.md` の dialogues 生成ルール参照）。
  既存セクションに前置きが残っている場合、改善対象に含まれていれば除去する。
```

## 既存55セクションのリカバリ

新プロンプトを反映しても、既に DB に入っている前置き付きセクションは自動では消えない。対処の選択肢:

| 戦略 | 概要 | コスト | リスク |
|------|------|--------|--------|
| (1) 全セクション再生成 | 該当lessonを `lesson_generate` で作り直す | 高（TTS再生成も） | 既存の良い箇所が変わってしまう |
| (2) lesson_improver でセクション単位修正 | annotation で「前置き除去」を指示し対象セクションだけ再生成 | 中 | 段階的、安全 |
| (3) 後処理スクリプトで先頭文を機械的に除去 | dialogues[0].content / tts_text の先頭文をパターンマッチ削除 | 低 | TTS音声の再生成が必要・誤検出リスク |
| (4) 何もしない（新規生成からだけ適用） | 既存はそのまま | ゼロ | 既存lessonは不自然なまま残る |

**推奨: (4) → 必要に応じて (2)**。Lesson #1（バイブコーダー向けセキュリティ講座）はまだ試聴段階なので、配信フィードバック次第で improver による部分修正を判断する。一括スクリプト処理（3）はTTS再生成コストと誤検出リスクが見合わない。

## 実装ステップ

1. `prompts/lesson_generate.md:232` を **案A** の文言に書き換え
2. `prompts/lesson_improve.md` のルール末尾に前置き禁止を追記
3. 効果検証:
   - テスト用に 1 セクション分だけ `lesson_generate` を走らせ、`dialogues[0].content` を確認
   - キーワード（画面を見/画面に/読み上げ等）が含まれていないことを目視
   - `display_text` の中身は依然として全文読まれていることを確認（要件Aの維持）
4. 検証OKなら DONE.md に追記、TODO.md から該当行を削除
5. 既存セクションへの遡及対応は判断保留（上記表の戦略4）

## 検証コマンド（参考）

```bash
# 既存セクションの前置き出現率を再計測
python3 - << 'EOF'
import sqlite3, json
con = sqlite3.connect('data/app.db')
cur = con.cursor()
cur.execute("SELECT lesson_id, title, dialogues FROM lesson_sections WHERE dialogues IS NOT NULL AND dialogues != ''")
patterns = ['画面を見', '画面に', '読み上げ', '読みます', '見てね', '見てください', '読むね', '画面を読']
total = hits = 0
for lesson_id, title, dj in cur.fetchall():
    try:
        d = json.loads(dj)
    except Exception:
        continue
    if not d:
        continue
    total += 1
    if any(p in d[0].get('content','') for p in patterns):
        hits += 1
print(f'{hits}/{total} = {hits*100//max(total,1)}%')
EOF
```

新規生成後に再実行して新プロンプトの効果を確認する。

## リスク

- **要件Aが失われる可能性**: 前置きを禁止したことで LLM が "省略" や "要約" に走らないか。案Aで `display_text` 全文読み上げの強調を維持しているので低リスクだが、生成サンプルで必ず確認する
- **間延び対策の喪失**: 元の前置きは「読みますよ」と視聴者に注意を促す機能もあった。これが消えても字幕同時表示でカバーできるはず（broadcast.html で字幕は出ている）
- **改善ループでの再注入**: lesson_improver は前置きを残すまたは増やす可能性があったが、`lesson_improve.md` 側にも禁止を入れることで防ぐ
