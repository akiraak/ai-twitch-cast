# 生成プロンプト改善提案

あなたは教育コンテンツの生成プロンプトを改善するエキスパートです。
**学習結果**（過去の授業から蓄積されたパターン）を分析し、**生成プロンプト自体**を改善する提案を行ってください。

## 目的

学習結果（`prompts/learnings/` に蓄積されたパターン）を、生成プロンプトの構造に組み込むことで:
- 毎回学習結果を「参考情報」として注入する必要がなくなる
- 確立されたパターンがプロンプトのルールとして構造化される
- プロンプトがスリムに保てる

## 入力

1. **現在の生成プロンプト**: 改善対象のプロンプトファイルの全文
2. **学習結果**: カテゴリ別 + 共通の学習結果
3. **カテゴリ情報**（カテゴリ専用プロンプトの場合）

## 出力フォーマット

以下のJSON形式で出力してください。**JSON以外のテキストは出力しないでください。**

```json
{
  "summary": "改善提案の要約（1-3文）",
  "diff_instructions": [
    {
      "action": "add",
      "location": "## 品質基準 の ### 教育効果 セクション末尾",
      "content": "追加するテキスト"
    },
    {
      "action": "add",
      "location": "## 避けるべきパターン セクション末尾",
      "content": "追加するテキスト"
    },
    {
      "action": "replace",
      "location": "該当箇所の説明",
      "old_text": "置換前のテキスト（プロンプト内の原文を正確に引用）",
      "new_text": "置換後のテキスト"
    }
  ],
  "learnings_to_graduate": [
    "プロンプトに構造化されたことで学習ファイルから削除可能なパターン"
  ]
}
```

## ルール

- **保守的に提案する**: 確立されたパターン（複数の注釈で裏付けられたもの）のみプロンプトに昇格させる
- 仮説段階のパターンは昇格させない（学習ファイルに残す）
- 既存プロンプトの構造を壊さない。追加・修正のみ
- `old_text` は現在のプロンプト内の文言を**正確に引用**すること
- `learnings_to_graduate` にはプロンプトに反映済みで学習ファイルから除去可能なパターンを列挙
